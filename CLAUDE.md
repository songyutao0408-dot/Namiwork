# 项目说明（写给 Claude Code）

## 项目是什么

本仓库用于开发一个**纳米Work的案例/产品展示页**，面向外部宣传使用（潜在客户、渠道伙伴、投资人等可能会打开这个页面）。目标是让不了解纳米Work的人，通过这个页面快速理解"纳米Work是什么、能帮谁解决什么问题、有哪些真实价值"。

## 在开始任何开发之前

请先阅读 `docs/纳米Work产品背景.md`，这是关于纳米Work的产品定位、核心机制、安全能力和真实用户案例的背景资料。这份文档是唯一权威的产品事实来源——页面文案、功能描述、案例引用都应基于这份文档，不要凭空编造或过度演绎纳米Work没有的功能。

如果 `docs/` 目录下之后新增了其他资料文件，也请一并阅读。

## 内容原则

- **对外宣传属性**：语气可以有营销感、有信服力，但事实类描述（功能、数据、案例）要忠于背景资料，不夸大、不编造具体数字。
- **案例部分**：背景资料里的真实用户案例数据来自用户真实使用记录，涉及具体企业/个人信息的地方注意脱敏或使用背景资料里已经处理过的表述方式，不要补充资料中没有的具体身份信息。
- **不需要**引入竞品对比内容（腾讯WorkBuddy、阿里悟空、字节ArkClaw等）——这个仓库的资料里不包含这些，也不应主动添加。

## 技术栈

没有强制要求，由你根据页面复杂度自行判断和选择（比如纯静态 HTML/CSS/JS，或者 React 等）。原则：

- 优先选择能快速搭建、易于部署（如 GitHub Pages / Vercel 等静态托管）的方案，除非确有交互复杂度需要引入框架。
- 如果引入构建工具或框架，请在 README 里写清楚本地开发和构建部署的命令。

## 设计方向建议（可参考，非强制）

背景资料中可以提炼的几个适合作为页面视觉/结构重点的差异化卖点：

1. **"一人成军，百事可托"** —— 核心Slogan，适合做首屏主视觉
2. **三类核心人群分层展示** —— 中小企业主 / OPC创业者 / 职场精英，各自的场景和价值，适合做成分区或Tab切换
3. **"动态网页交付"能力** —— 区别于传统AI工具"只给答案"，这是最大的差异化亮点，可以考虑用可视化/动效的方式呈现"生成 → 验收 → 分享 → 更新"这个流程
4. **安全信任背书** —— 纳米卫士三道防线，适合放在页面靠后位置做信任加固
5. **真实案例** —— 可以做成卡片墙或轮播展示
